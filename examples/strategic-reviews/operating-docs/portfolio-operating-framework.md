# Portfolio Operating Framework

## Purpose

The operating framework helps the operator manage the whole portfolio without
flattening it into tasks. It separates the human-facing surface from the
backend sources and keeps the shape of the work visible enough to choose the
next sane move.

## Operating Stack

1. North Star: why, constraints, principles, tradeoffs.
2. Quarterly compass: the live priorities for the current quarter.
3. Entity briefs: current state, outcome, owners, sources, blockers.
4. Operating work map: synthesis across trackers, notes, and source systems.
5. Execution trackers.
6. Session plan.

## Human Surface Contract

The operator should only need to use the human-facing surface (notes, the work
board, and agent chat). Agents may use a richer backend model, but the output
should be short briefs, clear choices, and humane next moves.
